

# Skill System

This folder contains reusable AI collaboration knowledge collected during development of ASCII Camera and related interactive media projects.

Skills are not tutorials.
Skills are not framework documentation.

Skills are:
- production memory
- runtime knowledge
- interaction philosophy
- browser behavior knowledge
- AI workflow constraints
- visual identity preservation
- historical debugging lessons

---

# Structure

Skills are divided into:

- project-specific skills
- browser/runtime skills
- interaction and animation skills
- AI workflow skills
- reusable media-system knowledge

# Philosophy

## Runtime behavior is the real source of truth

Generated code may appear correct while:
- rendering is unstable
- media timing is incorrect
- browser behavior differs
- deployment behavior changes

Real runtime behavior matters more than static code quality.

---

## AI naturally modernizes interfaces

AI models naturally drift toward:
- clean modern UI
- generic mobile-app patterns
- minimal dashboard aesthetics
- over-optimized interaction systems

Skills exist partly to preserve:
- retro atmosphere
- experimental UI personality
- machine-like pacing
- computational texture
- imperfect visual identity

---

## Browser environments are unstable systems

Interactive media applications are affected by:
- browser rendering differences
- GPU acceleration behavior
- embedded runtime environments
- autoplay restrictions
- iframe behavior
- camera pipeline differences

Stable runtime behavior requires continuous real-browser testing.

---

# Long-Term Goal

The long-term goal is to build:
- reusable browser/runtime knowledge
- reusable interaction philosophy
- reusable media-system workflows
- reusable AI collaboration memory

Future projects should inherit runtime and interaction lessons instead of rediscovering them repeatedly.
